# Velunee - Final Potential Improvement Outline

## User-Perspective Product Requirements and Future Instructions

This document defines how Velunee should evolve into a useful, trusted and differentiated personal AI companion. It focuses on what users should experience, not technical implementation.

---

# 1. Product North Star

Velunee should not feel like a collection of unrelated tools.

It should feel like:

> **One intelligent companion that understands the user's situation, connects the important parts of their life and helps them decide what to do next.**

Every feature should support at least one of these outcomes:

- Save the user time
- Reduce confusion
- Improve daily decisions
- Increase confidence
- Protect privacy
- Support progress toward personal goals

Do not add features only because competing apps have them.

---

# 2. Strongest user promise

The main product message should be:

> **Tell Velunee what is happening in your life. It will understand the situation, consider what matters to you and help you choose the best next step.**

A shorter promotional version:

> **One companion. Smarter everyday decisions.**

Velunee's advantage should not be "we have many features."

Its advantage should be:

> **Our features understand and support each other.**

---

# 3. Core improvement priorities

Velunee should improve in this order:

1. Create one intelligent and useful home experience
2. Make personalization visible and controllable
3. Build a strong decision-assistance system
4. Allow users to take action from advice
5. Add trust, transparency and verification
6. Improve privacy and community safety
7. Connect finance, weather, wardrobe, study and planning
8. Improve speed, simplicity and international usability
9. Add new features only after the core experience is reliable

---

# 4. Create one intelligent home experience

## Current risk

Users may feel that Velunee contains separate sections for chat, weather, money, study, wardrobe and community, but does not truly connect them.

## Required improvement

The home screen should become a personal command centre.

It should answer:

- What matters today?
- What should I pay attention to?
- What is my next important action?
- Is there anything I may forget?
- What decision can Velunee help me make?

## Recommended home content

The home screen may show:

- Greeting
- Weather and useful weather advice
- Today's schedule
- Important reminders
- Safe-to-Spend amount
- Upcoming bill
- Current goal progress
- Study or work priority
- One personalized recommendation
- Continue recent conversation
- Quick action buttons

## Example

> **Good morning, Sara.**  
> Rain is expected before you leave work.  
> Your first meeting starts at 9:30 AM.  
> You can safely spend approximately USD 28 today.  
> Your electricity bill is due tomorrow.  
> Would you like an outfit suggestion for today's weather?

The user should be able to control which cards appear.

# 5. Velunee Daily Brief

Create an optional personalized daily summary.

## The brief may include

- Weather
- Calendar
- Reminders
- Money Weather
- Safe-to-Spend
- Goal progress
- Study plan
- Habit or wellbeing reminder
- Important community activity
- One useful suggestion

## User controls

Users should choose:

- Morning or evening brief
- Delivery time
- Included topics
- Notification or in-app only
- Detailed or short version

Avoid generic messages such as:

> Have a wonderful day.

Prefer:

> Rain is expected at 5 PM. Carrying an umbrella may help you avoid an unplanned ride-share expense.

# 6. Velunee Decide - signature decision system

Velunee Decide should become the main feature that separates Velunee from ordinary AI chatbots.

Users should be able to ask:

- What should I wear?
- Can I afford this?
- Which option is better?
- Should I buy this or wait?
- Should I go out today?
- What should I study first?
- How should I prepare for tomorrow?
- Which meal is better for my time and budget?
- Should I accept this plan?
- What is the best next step?

## Every decision response should contain

### 1. Recommended option

Give one clear recommendation.

### 2. Why it was recommended

Explain the main reasons simply.

### 3. Important information considered

For example:

- Weather
- Budget
- Schedule
- Preferences
- Existing wardrobe
- Previous choices
- Goal deadlines
- User-defined priorities

### 4. Alternative option

Provide another reasonable choice.

### 5. Possible impact

Explain what may happen if the user chooses each option.

### 6. Next action

Offer one simple action the user can take immediately.

## Example

> **Best option:** Wear your navy outfit and carry a light jacket.
>
> **Why:** It matches your office preference, the afternoon temperature will be lower, and you already selected this outfit as comfortable.
>
> **Alternative:** The beige outfit is suitable if you prefer a more formal look.
>
> **Next step:** Save the navy outfit for today.

The user must always remain in control.

- Make temporary
- Keep private
- Disable future use

This improves trust and makes personalization feel real.

- Add to calendar

- Add item to wardrobe

# 10. Improve conversational understanding

Users should not need to write perfect prompts.

Velunee should understand short, natural messages.

Examples:

> Tomorrow office, what should I wear?

> Can I buy this phone?

> I spent 20 on lunch.

> Explain photosynthesis for class 6.

> I feel confused about tomorrow.

Velunee should identify:

- User intent
- Missing information
- Relevant personal context
- Required tools or features
- Whether confirmation is needed

## Clarification rule

Ask only when the missing information could significantly change the result.

Do not repeatedly ask unnecessary questions.

# 12. Trust and verification system

Velunee should distinguish between factual information, personal advice and calculations.

## For factual answers

Show when appropriate:

- Sources
- Date checked
- Confidence
- Verify again
- Possible uncertainty

## For financial calculations

Show:

- Inputs used
- Formula
- Exchange rate
- Rate timestamp
- Assumptions
- Included accounts
- Excluded accounts
- Edit values

## For recommendations

Show:

- Why it was suggested
- Information considered
- What was assumed
- Alternative option

## For educational answers

Show:

- Explanation level
- Subject and grade
- Step-by-step reasoning
- Practice questions
- Sources where appropriate

Velunee must never pretend to be certain when information is incomplete.

# 29. First-use onboarding

The onboarding should not ask too many questions at once.

## Recommended onboarding steps

### Step 1: Main purpose

Ask:

> What would you like Velunee to help with first?

Options:

- Daily life
- Money
- Study
- Style
- Planning
- Community
- Explore everything

### Step 2: Basic preferences

Ask for:

- Name or preferred name
- Language
- Country or region
- Time zone

### Step 3: Personalization permission

Explain:

> Velunee can remember selected preferences to make future help more useful. You control what is saved.

### Step 4: First useful action

Immediately help the user complete one meaningful task.

Examples:

- Create today's plan
- Check weather
- Add first expense
- Ask a study question

Do not require the user to complete every setup section before using the app.

Do not show every feature on the first screen.

Introduce features when they become relevant.

Example:

> Velunee Balance can also show how this purchase affects your monthly plan. Would you like to set it up?

This feels more natural than displaying dozens of options at once.

# 42. Instructions for future product and coding work

Use the following instructions whenever improving Velunee:

> Improve Velunee as one connected personal AI companion rather than a collection of separate tools. Every new feature must solve a clear user problem, connect with the wider assistant when permission is granted and provide a visible benefit.
>
> Prioritize the intelligent Home experience, Velunee Daily, Velunee Decide, user-controlled memory, transparent recommendations, direct actions, privacy, international support and community safety.
>
> Keep the interface simple. Do not overload screens or onboarding. Introduce advanced features progressively.
>
> AI-generated advice must clearly separate facts, calculations, assumptions and personal recommendations. Important financial calculations must come from a deterministic system and include "How was this calculated?"
>
> Users must be able to view, edit and delete remembered information. Financial information, private chats, uploaded images and personal memories must never be shared with the community automatically.
>
> Maintain useful behaviour when the AI service, internet connection or external integration is unavailable. Show clear loading, retry, offline and error states without losing user input.
>
> Do not prioritize investment trading, lending, dating, live streaming, public financial rankings or other distracting features until the core companion experience is stable, trusted and genuinely useful.
>
> Evaluate every improvement from the user's perspective: Does it save time, reduce confusion, improve a decision, protect privacy or help the user make progress?

# 43. Final product positioning

Velunee should ultimately be described as:

> **Velunee is a personal AI companion that understands your daily life, connects your plans, money, learning, style and goals, and helps you confidently decide what to do next.**

The strongest future version of Velunee is not the app with the most features.

It is the app where users feel:

> **Velunee understands what matters to me, explains its advice, protects my privacy and helps me take the next step.**
